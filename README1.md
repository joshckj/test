# backend_gasleakagent

A FastAPI backend for SP Digital's Gas Leak Prediction Agent.
Accepts natural language messages via a single `/chat` endpoint, routes them
through a Qwen-powered LangChain agent, and executes the appropriate skill —
either running a gas leak risk prediction or retraining the CatBoost model.

## ⚠️ Important — db/static/ is Temporary

`app/db/static/` is a **runtime-only** folder. It is in `.gitignore` and must never be committed.

| File | Source | Lifetime |
|---|---|---|
| `gasleakmodel.cbm` | Downloaded from Azure Blob on startup | Until server restarts |
| `Prediction-<Q><Y>.csv` | Generated by analysis_map skill | Until next analysis |
| `<Q>_<Y>.html` | Generated by analysis_map skill | Until next analysis |

**The model is the source of truth in Azure Blob Storage** — `db/static/` is just a local cache for the current server session.

## How It Works

```
Server starts
      ↓
download_model() pulls gasleakmodel.cbm from Azure Blob → db/static/
      ↓
Frontend sends POST /chat { "message": "run analysis for Q1 2025" }
      ↓
app/api/routes.py receives message
      ↓
app/gasleakagent/agent.py passes message to Qwen via LangChain
      ↓
Qwen decides which skill to call
      ↓
Skill fetches data from PostgreSQL (no local CSVs)
      ↓
Skill uses model from db/static/gasleakmodel.cbm
      ↓
Saves CSV + HTML map to db/static/
      ↓
Returns { "response": "..." } to frontend
      ↓
Frontend calls GET /map/<filename> to display hex map
Frontend calls GET /download/<filename> to download CSV
```

## API Reference

### POST /chat
Send a message to the agent.

**Request:**
```json
{ "message": "Run analysis for Q1 2025" }
```

**Response (success):**
```json
{ "response": "Q1 2025 analysis complete. High: 4, Medium: 11, Low: 23, Minimal: 108." }
```

**Response (error):**
```json
{ "detail": "error message" }
```

---

### GET /map/{filename}
Serve the generated Folium HTML hex map to the frontend UI.

**Example:**
```
GET /map/Q1_2025_25_15_5.html
```

Returns the full HTML map for embedding in the frontend.

---

### GET /download/{filename}
Download the prediction CSV.

**Example:**
```
GET /download/Prediction-Q12025.csv
```

Returns the CSV file as a download.

---

### GET /health
Check if the server is running.

**Response:**
```json
{ "status": "ok" }
```

## Skills

### analysis_map
Fetches data from PostgreSQL, predicts gas leak risk per hex cell, generates
a risk-classified CSV and interactive HTML map saved to `db/static/`.

| | |
|---|---|
| **Triggered by** | "run analysis", "predict risk", "show map", "Q1 2025", "high risk areas" |
| **Inputs** | quarter (Q1–Q4), year, risk thresholds (default High=25, Medium=15, Low=5) |
| **Data source** | PostgreSQL — `dmis_main_leaks`, `pipe_data_files` tables |
| **Model** | `db/static/gasleakmodel.cbm` (downloaded from Azure Blob on startup) |
| **Outputs** | `db/static/Prediction-<Q><Y>.csv`, `db/static/<Q>_<Y>_<H>_<M>_<L>.html` |

### retrain_model
Fetches data from PostgreSQL, retrains the CatBoost model, saves to
`db/static/`, then uploads back to Azure Blob Storage.

| | |
|---|---|
| **Triggered by** | "retrain", "update model", "new data added", "train again" |
| **Inputs** | None — fetches directly from PostgreSQL |
| **Data source** | PostgreSQL — `dmis_main_leaks`, `pipe_data_files` tables |
| **Output** | `db/static/gasleakmodel.cbm` + uploaded to Azure Blob |

## Data Sources

| Data | Where it lives | How it's accessed |
|---|---|---|
| Gas leak records | PostgreSQL — `dmis_main_leaks` | `app/db/queries.fetch_leaks()` |
| Pipe asset data | PostgreSQL — `pipe_data_files` | `app/db/queries.fetch_pipes()` |
| Trained model | Azure Blob Storage | Downloaded to `db/static/` on startup |
| Hex map output | `db/static/` (runtime only) | Served via `GET /map/<filename>` |
| Prediction CSV | `db/static/` (runtime only) | Downloaded via `GET /download/<filename>` |

## Creating a New Skill

1. Copy the template:
```bash
cp -r app/skills/template app/skills/your_skill_name
```

2. Fill in `SKILL.md`

3. Write your pipeline in `scripts/main.py`

4. Register in `app/gasleakagent/agent.py`:
```python
@tool
def your_skill_name(param: str) -> str:
    """Describe what this skill does so the LLM knows when to call it."""
    result = subprocess.run(
        ["python3", "app/skills/your_skill_name/scripts/main.py"],
        capture_output=True, text=True,
        cwd=ROOT_DIR
    )
    return result.stdout or result.stderr

tools = [analysis_map, retrain_model, your_skill_name]
```

## Setup

### 1. Navigate to project root
```bash
cd backend_gasleakagent
```

### 2. Create virtual environment
```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
```

### 4. Set up environment variables
```bash
cp .env.example .env
# Fill in all values
```

### 5. Start the server
```bash
chmod +x start.sh    # only needed once
./start.sh
```

Server runs at `http://localhost:8000`

## Common Errors

| Error | Cause | Fix |
|---|---|---|
| `ModuleNotFoundError: core.config` | Wrong working directory | Run from `backend_gasleakagent/` root |
| `ModuleNotFoundError: backend_gasleakagent` | Wrong import path | Change to `from core.config import ...` |
| `FileNotFoundError` on model | Model not in `db/static/` | Check Azure Blob connection and startup download |
| `OperationalError` on DB | PostgreSQL not reachable | Check `DB_URL` in `.env` |
| `FileNotFoundError` on scripts | Wrong `cwd` in subprocess | Set `cwd=ROOT_DIR` in `agent.py` |
| `ValueError: all targets equal` | Bad leaks data | Check `dmis_main_leaks` table has valid records |
| `uvicorn: command not found` | Not in venv | Run `source .venv/bin/activate` first |

## Notes
- Always run from `backend_gasleakagent/` root
- `app/db/static/` is runtime-only — never commit it
- Model source of truth is Azure Blob — `db/static/` is just a local cache
- Shapefiles in `assets/` are static — do not modify unless boundaries change
- After retraining, model is auto-uploaded to Azure Blob and available immediately
- `app/core/llm.py` is where Qwen is configured — change model name there
- `catboost_info/` is auto-generated during retraining — already in `.gitignore`
